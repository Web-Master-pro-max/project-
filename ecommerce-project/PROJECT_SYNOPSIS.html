📋 Project Overview

ShopHub is a comprehensive e-commerce platform built with modern web technologies. It provides a complete solution for online retail operations, including product management, shopping cart functionality, order processing, and user authentication with role-based access control.

Target Users: This project serves as an ideal final-year project for BCA (Bachelor of Computer Applications) students, demonstrating full-stack web development capabilities including database design, backend API development, and frontend user interface implementation.

🗄️ Database Schema

Table Purpose Key Fields
users User accounts and profiles id, name, email, password, role, created_at
products Product catalog id, name, description, price, stock, category, image_url
cart Shopping cart items id, user_id, product_id, quantity
orders Customer orders id, user_id, total_amount, status, created_at
order_items Items in each order id, order_id, product_id, quantity, price

🔌 REST API Endpoints

Authentication

POST /api/auth/register - Register new user
POST /api/auth/login - User login
POST /api/auth/logout - User logout
GET /api/auth/check - Check authentication status
GET /api/auth/me - Get current user info

Products

GET /api/products - Get all products with filters
GET /api/products/:id - Get single product
POST /api/products - Create product (Admin)
PUT /api/products/:id - Update product (Admin)
DELETE /api/products/:id - Delete product (Admin)
GET /api/products/categories/all - Get all categories

Shopping Cart

GET /api/cart - Get user's cart
POST /api/cart - Add item to cart
PUT /api/cart/:productId - Update cart item
DELETE /api/cart/:productId - Remove from cart
DELETE /api/cart - Clear entire cart

Orders

GET /api/orders - Get user's orders
GET /api/orders/:id - Get order details
POST /api/orders - Create new order
PUT /api/orders/:id - Update order status (Admin)

🔐 Authentication & Security

Session Management

  • 7-day session expiration with automatic refresh
  • 5-minute periodic server-side validation
  • Activity-based automatic session refresh
  • HTTPOnly cookies prevent XSS attacks
  • SameSite=Lax cookie policy

Password Security

  • bcryptjs hashing with 10 rounds
  • Minimum 6 character passwords enforced
  • Passwords never stored in plain text

Authorization

  • Role-based access control (Admin/Customer)
  • Protected routes require authentication
  • Admin endpoints check user role

🚀 Setup & Deployment

Prerequisites

  • Node.js v24.13.1 or higher
  • MySQL 8.0 or compatible
  • npm (Node Package Manager)

Installation Steps

1. Clone/Extract project
2. Install dependencies: npm install
3. Configure .env file with database credentials
4. Run database setup: node setup-database.js
5. Start server: npm start (or npm run dev for development)

Default Credentials

User Type Email Password
Admin admin@example.com admin123
Customer john@example.com customer123

Access Points

  • Homepage: http://localhost:3000/
  • Products: http://localhost:3000/products
  • Login: http://localhost:3000/login
  • Register: http://localhost:3000/register
  • API Base: http://localhost:3000/api/

⚡ Performance & Optimization

  • Express compression middleware for gzip compression
  • Rate limiting (100 requests per 15 minutes)
  • Helmet.js for security headers
  • CORS configuration for cross-origin requests
  • Database connection pooling via Sequelize
  • Efficient session management with automatic refresh
